# Use Case: View Profile

## Goal

An authenticated user views their full account information and platform settings on the profile page, and can update their platform settings from the Platform Settings tab.

## Actor

Authenticated User — a guest who has completed the login flow and has an active
session in the Zustand store.

## Screen

- **Route:** `/profile`
- **Page:** `ProfilePage`
- **Entry point:** Authenticated user clicks the Profile link in the sidebar.

## Trigger

Authenticated user clicks the Profile nav link in the sidebar or navigates
directly to `/profile`.

## Domain Models

- `domain/model/session`
- `domain/model/user`
- `domain/model/user-settings`

## Flows

- `domain/flows/user-session` (Flow B — View Profile Page)
- `domain/flows/update-user-settings`

## Happy Path

### Tab: Profile Information (existing)

1. Authenticated user clicks Profile in the sidebar.
2. Frontend reads the full user profile from the Zustand session store.
3. Profile page renders: `firstName`, `lastName`, `email`, `address`, `status`, `createdAt` (formatted as local date), and the current date.

### Tab: Platform Settings (new)

1. Authenticated user clicks the Platform Settings tab on the Profile page.
2. Frontend renders the General Platform Settings section.
3. A labelled dropdown for "Market Data Feed" is displayed, pre-populated with the user's current `feedType` from the Zustand session store (`session.settings.feedType`).
4. The dropdown shows two options:
   - **Synthetic Feed** — tooltip: "Market data generated by the platform, used for testing and playing around."
   - **Real Market Feed** — tooltip: "Market data obtained from real trading platforms."
5. User selects an option from the dropdown.
6. Frontend immediately sends `PATCH /api/v1/users/{userId}/settings` with `{ "feedType": "<selected value>" }`.
7. On HTTP 200 response, frontend updates `session.settings` in the Zustand store with the returned settings object.
8. Dropdown reflects the saved value. A brief "Saved" confirmation is shown.

## Failure Scenarios

| Scenario | Outcome |
|---|---|
| No active session | User is redirected to `/login`. |
| PATCH returns HTTP 400 | Inline error shown on the dropdown. Dropdown reverts to previous value. |
| PATCH returns HTTP 401 | User is redirected to `/login`. |
| PATCH returns HTTP 403 | Generic error message shown. |
| PATCH returns HTTP 500 | Generic error message shown. Dropdown reverts to previous value. |

## Out of Scope

- Editing profile fields (firstName, lastName, address, email).
- Uploading a profile photo.
- Deleting the account.
- Real market feed data integration (stored as a preference only in this iteration).
